# PDF Page Remover — Stitch Homepage Prompt

Use this prompt to generate the high-fidelity homepage concept for the PDF Page Remover project.

```text
Design a high-fidelity homepage concept for PDF Page Remover — a single-function PDF utility that only removes selected pages from a PDF.

Product positioning
- Product: PDF Page Remover
- Core flow: Upload PDF → Select pages to delete → Download result
- Audience: office workers, students, freelancers, and light document users
- Tone: clean, trustworthy, practical, lightweight
- The page should feel like a focused document-processing tool, not a marketing-heavy SaaS landing page or an all-in-one PDF platform

Hard constraints
- Do not add merge, split, compress, OCR, sign, convert, edit, or any other extra PDF tools
- Do not add Pro / Upgrade / Subscription / Enterprise language
- Do not use emoji icons or decorative 3D visuals
- Avoid generic SaaS dashboard styling
- Keep compliance and limits visible, but secondary to the main workflow

Primary goal
- Make the page feel like a real utility workspace
- Keep the main path obvious and linear
- Make page selection the visual center of the experience
- Show trust and limits without overpowering the tool flow

Homepage structure

1. Hero / upload area
- Compact brand strip at the top
- Large upload dropzone or tool card
- Main CTA: Upload PDF
- Short supporting line: upload, select pages, and download the cleaned file
- The hero should feel like a tool screen, not a marketing hero

2. Tool preview / page selection area
- Show a realistic PDF preview card
- Use compact page thumbnails in a grid
- Each thumbnail should show:
  - page number
  - selected / unselected state
  - delete mark or badge when selected
- Above the thumbnails, show a summary line such as:
  - “3 pages selected for deletion”
- Make the delete action obvious but not aggressive

3. Action area
- Primary CTA: Delete selected pages
- Secondary CTA: Clear selection
- Keep one dominant action at a time

4. Processing and success states
- Processing state: subtle loader and concise status text like “Removing selected pages...”
- Success state: ready-to-download result card
- Primary CTA in success state: Download result

5. Limits and trust
- Show file limits in a compact, readable block
- Free: up to 20MB and 200 pages
- Max: up to 100MB and 500 pages
- Password-protected or encrypted PDFs are not supported
- Anonymous use available
- Credits may be required for heavier jobs
- Keep this area practical and neutral, not salesy

6. Compliance
- Include Privacy, Terms, and Cookie links
- Analytics off by default
- Cookie consent visible but lightweight
- Keep compliance readable but quiet

7. FAQ link
- Add a clear link to /faq
- It should feel like support, not a CTA

Visual direction
- Light background, dark text
- Soft borders, subtle shadows, restrained radius
- One stable accent color for the main CTA and active states
- Strong document-utility feel with low visual noise
- Clean, professional, and credible

States to show clearly
- Empty upload state
- Uploading state
- Scanning state
- Preview ready state
- Page selected state
- Processing state
- Success / download state
- Over-limit state
- Encrypted / unsupported state
- Credit insufficient state
- Cookie consent state

Layout notes
- Desktop: simple left-to-right structure
- Mobile: keep the primary CTA above the fold
- The preview and selection UI should remain usable on narrow screens
- Compliance should never block the main workflow

Final requirement
- The result should feel like a trustworthy, focused PDF page-removal utility
- The main path must stay singular and obvious
- Avoid adding any unsupported features or extra product directions
```
