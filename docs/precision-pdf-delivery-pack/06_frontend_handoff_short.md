# Precision PDF — Short Front-End Handoff

## Product
Single-function PDF page remover.

## Core flow
Upload PDF → select pages to remove → delete selected pages → download result

## Homepage
- Hero: `Remove PDF pages without friction`
- Subheadline: `Upload a PDF, select the pages you want removed, and download the cleaned file.`
- Primary CTA: `Upload PDF`
- Secondary CTA: `See workspace`
- Blocks: hero, workspace preview, 3-step workflow, limits/FAQ, final CTA

## Workspace
- Same visual system as homepage
- Hero title matches homepage tone
- Page thumbnail grid with selected/unselected state
- `Select all`, `Clear selection`, `Invert selection`
- Primary action: `Delete selected pages`
- Success state: download-ready result card + `Start over`

## Visual rules
- White/slate background
- Blue accent only for actions and selected state
- Rounded cards, subtle borders, soft shadows
- Tool-first, not marketing-first

## Copy rules
- Keep copy short and direct
- Avoid broad claims and avoid legacy generator language
- Keep privacy/usage details lightweight

## Acceptance
- New user understands the product in under 5 seconds
- Next action is always clear
- Build passes
- No legacy generator references remain in product-facing UI
